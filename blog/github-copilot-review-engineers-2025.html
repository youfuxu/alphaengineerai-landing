What Is GitHub Copilot?

Copilot is an AI pair programmer that lives inside your editor as an extension. It autocompletes code as you type, suggests whole functions from a comment, and — in its Chat mode — answers questions about your code and proposes fixes. Because it's a GitHub/Microsoft product, it's deeply integrated with VS Code and ships across every major IDE, which is a big part of why it's the default for so many engineers.

The mental model: Copilot is autocomplete on steroids that has read a lot of code. It's not trying to rebuild your editor or take over whole tasks — it's trying to make the editor you already love quietly faster. For a lot of engineers, that's exactly the right amount of AI.

Key Features That Matter

1. In-Editor Completion

The core feature, and still the best part. As you type, Copilot offers ghost-text suggestions — sometimes a line, sometimes a whole function — that you accept with Tab. For boilerplate, repetitive patterns, and "I know what I want but typing it is tedious" moments, it's a genuine speed boost that compounds over a workday.

[ Screenshot: Copilot ghost-text completing a function from a comment ]

2. Copilot Chat

The chat sidebar lets you ask about your code, explain errors, generate tests, and request fixes without leaving the editor. It's not as deep on whole-repo reasoning as dedicated AI editors, but for "explain this regex" or "write a test for this function," it's right there and fast.

[ Screenshot: Copilot Chat explaining an error and proposing a fix ]

3. Broad IDE Support

This is Copilot's quiet superpower. It works in VS Code, JetBrains IDEs, Neovim, and Visual Studio. If your workflow is built around a specific editor, you don't have to abandon it to get AI assistance — Copilot meets you where you already are. For engineers who are particular about their setup, that matters more than any single feature.
